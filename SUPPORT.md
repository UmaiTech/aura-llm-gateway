# Getting Support

Thank you for using Aura LLM Gateway! Here's how to find help, depending on what you need.

## Documentation first

Most common questions are already answered in the docs:

- **[README.md](README.md)** — install, run, configure
- **[docs/](docs/)** — architecture, API reference, provider mapping
- **[CONTRIBUTING.md](.github/CONTRIBUTING.md)** — development setup
- **[CLAUDE.md](CLAUDE.md)** — project conventions
- **[Open Responses API spec](https://www.openresponses.org/specification)** — protocol reference

## Asking a question

Open a thread in
[GitHub Discussions](https://github.com/UmaiTech/aura-llm-gateway/discussions).
This is the right place for:

- "How do I…?" usage questions
- Configuration and deployment help
- Ideas, design discussion, RFCs
- Showing off what you built with Aura

Please search existing discussions before opening a new thread.

## Reporting a bug

Open an [Issue](https://github.com/UmaiTech/aura-llm-gateway/issues/new?template=bug_report.md)
using the bug-report template. A good bug report includes:

- The version of Aura you're running (`aura-proxy --version` or commit SHA)
- A minimal reproduction
- What you expected vs. what happened
- Any relevant logs (with `RUST_LOG=debug` if possible)
- The provider(s) involved, if it's a provider-specific issue

## Requesting a feature

Open an [Issue](https://github.com/UmaiTech/aura-llm-gateway/issues/new?template=feature_request.md)
using the feature-request template. Tell us **what problem you're trying
to solve**, not just the solution you have in mind — that helps us evaluate
alternatives.

## Reporting a security issue

**Do not open a public issue.** See [SECURITY.md](SECURITY.md) for the
private disclosure process. tl;dr: open a draft GitHub Security Advisory
or email **security@umai.tech**.

## Commercial support

Umai Tech offers managed deployments and commercial support. Reach out at
**hello@umai.tech** if you're interested.

## Response times

This is an open source project maintained on a best-effort basis. We aim
to triage new issues within a few business days, but please be patient.
Pull requests with reproductions and proposed fixes are very welcome and
typically get the fastest response.
